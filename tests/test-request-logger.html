RequestLogger Tests

Manual Test Procedures

These tests must be run in Firefox via about:debugging with the extension loaded:

1. Browse to a news site with many third-party scripts (e.g., cnn.com)
   → Wait 4+ seconds for at least 2 batch cycles
   → Check about:debugging > Storage > Extension Storage
   → "thirdPartyRecords" key should contain attribution records

2. Verify request type filtering
   → Open Network tab, note request types
   → Only script, xmlhttprequest, beacon, ping, sub_frame should appear in records
   → image, font, stylesheet requests should NOT appear

3. Deduplication test
   → Load a page with many requests to the same third-party domain
   → Verify one record per unique {thirdPartyDomain, cookieName, cookiePath} combo
   → Not one record per request

4. Persistence test
   → Browse some sites, wait for batch to process
   → Restart the extension via about:debugging
   → Run in background console: RequestLogger.getThirdPartyRecords()
   → Should return previously saved records

5. Clear test
   → Run in background console: RequestLogger.clearAllRecords()
   → Run: RequestLogger.getThirdPartyRecords()
   → Should return empty array
   → Check storage — "thirdPartyRecords" key should be removed

6. Verify batch logging
   → Watch background console for "[DomainGuard] Batch processed:" messages
   → Should appear every 2 seconds when there are requests to process
