Deletion Engine Tests

Manual Test Procedures

These tests must be run in Firefox via about:debugging with the extension loaded:

=== Content Script (storage-reader.js) ===

1. Read storage
   → Navigate to any site (e.g., walmart.com)
   → In the site's DevTools console: localStorage.setItem("test-key", "test-value")
   → In background console: get the active tab ID
     let tabs = await browser.tabs.query({active: true, currentWindow: true});
     let tabId = tabs[0].id;
   → Send read message:
     await browser.tabs.sendMessage(tabId, {type: "read-storage"})
   → Should return {localStorage: [{key: "test-key", value: "test-value"}, ...], sessionStorage: [...]}

2. Delete storage item
   → browser.tabs.sendMessage(tabId, {type: "delete-storage-item", storageType: "local", key: "test-key"})
   → Should return {success: true}
   → Verify in DevTools > Storage: item is gone

3. Clear all storage
   → Add some localStorage and sessionStorage items on the page
   → browser.tabs.sendMessage(tabId, {type: "clear-storage", storageType: "all"})
   → Should return {success: true}
   → Verify in DevTools > Storage: both are empty

4. Content script is inert on injection
   → Navigate to a new site, check console — no output from storage-reader.js
   → No DOM changes, no network requests

=== Deletion Engine (deletion.js) ===

5. Manual cookie deletion
   → Find a cookie: let cookies = await browser.cookies.getAll({domain: "walmart.com"})
   → Delete it: await Deletion.deleteCookie(cookies[0])
   → Verify gone: await browser.cookies.getAll({domain: "walmart.com"})
   → Watch for "[DomainGuard] Deleted cookie:" log message

6. Secure vs non-secure cookie URLs
   → Find a secure cookie (secure: true)
   → Delete it — URL in logs should use https://
   → Find a non-secure cookie (secure: false)
   → Delete it — URL in logs should use http://

7. Leading dot domain handling
   → Find a cookie with domain ".walmart.com"
   → Delete it — URL should use "walmart.com" (dot stripped)

=== Auto Delete-on-Leave ===

8. Toggle ON, close all tabs for a domain
   → Open walmart.com in a tab, wait for cookies to appear
   → Verify toggle: Deletion.getToggleState("walmart.com") → should be true (default)
   → Close all walmart.com tabs
   → Wait ~1 second (500ms TabTracker delay + deletion)
   → Check: await browser.cookies.getAll({domain: "walmart.com"}) → should be empty
   → Watch for "[DomainGuard] Auto-deleting data for walmart.com" log

9. Toggle OFF, close all tabs
   → Open a site, wait for cookies
   → await Deletion.setToggleState("example.com", false)
   → Close all tabs for that domain
   → Wait ~1 second
   → Cookies should still exist (NOT deleted)

10. Toggle persistence
    → await Deletion.setToggleState("example.com", false)
    → Restart extension via about:debugging
    → Deletion.getToggleState("example.com") → should be false

11. Third-party data NOT affected by auto-delete
    → Browse sites to accumulate third-party records
    → Close all tabs for a first-party domain
    → RequestLogger.getThirdPartyRecords() → should still have data

=== Broadcast Tests ===

12. Deletion broadcast
    → Open the sidebar or panel
    → Delete a cookie via background console: Deletion.deleteCookie(...)
    → Panel should receive deletion-complete message (check panel console)

13. Broadcast with no listeners
    → Close all panels/sidebar
    → Delete a cookie via background console
    → Should NOT throw an error (try/catch wraps the broadcast)

=== Message Router Tests ===

14. Get cookies via message
    → From panel console: browser.runtime.sendMessage({type: "get-cookies", domain: "walmart.com"})
    → Should return array of cookies

15. Toggle via message
    → browser.runtime.sendMessage({type: "get-toggle-state", domain: "test.com"})
    → Should return true (default)
    → browser.runtime.sendMessage({type: "set-toggle-state", domain: "test.com", enabled: false})
    → browser.runtime.sendMessage({type: "get-toggle-state", domain: "test.com"})
    → Should return false

16. Third-party records via message
    → browser.runtime.sendMessage({type: "get-third-party-records"})
    → Should return array of record objects

17. Clear all third-party via message
    → Browse sites to accumulate records
    → browser.runtime.sendMessage({type: "clear-all-third-party"})
    → browser.runtime.sendMessage({type: "get-third-party-records"})
    → Should return empty array
